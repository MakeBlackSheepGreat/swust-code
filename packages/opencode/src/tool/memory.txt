Search session/project/global memory using BM25 over markdown bodies.
Use this to recall content persisted previously: project memory,
session checkpoints, task narratives under sessions/<sid>/tasks/,
session notes, and global preferences.

Memory layout: <data>/memory/<scope>/<scope_id>/<key>.md
Scopes: global | projects | sessions | cc

Query guidelines:
- Queries are OR-joined and BM25-ranked. A document matches if it
  contains any query word, then ranking favors rarer and more repeated
  matched words.
- Prefer 1-3 distinctive terms: function names, task IDs, exact phrases
  from directives, rare error text, or uncommon file names.
- Avoid long generic descriptions. Use "permission deadlock" instead of
  "the config params database connection issue".
- Punctuation is stripped by tokenization. For literals such as URLs,
  ports, file paths, or command flags, search one distinctive token.

A hit is authoritative. If search returns a result, use it even when a
nearby phrasing returned nothing.

If search returns 0 results:
1. Retry with fewer and rarer terms.
2. For exact literals with punctuation, grep the memory directory.
3. For verbatim original wording, use the history tool, which keeps raw
   conversation messages.
Widen scope progressively: session -> project -> global -> history.

Actions:
- search: OR-joined BM25 query with optional scope, scope_id, and limit.

After search returns paths, use Read on the most relevant path to load
full content. Snippets are truncated.

Note: SWUST currently indexes global/projects/sessions memory. The cc
scope and frontmatter type filter are accepted for MiMo API compatibility
but are not indexed by SWUST core yet.
